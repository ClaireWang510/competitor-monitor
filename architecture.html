AgentState 数据结构

class AgentState(BaseModel):
    competitor_name: str        # 当前处理的竞品
    raw_items: List[RawItem]    # 采集到的原始条目
    analyzed_items: List[...]   # LLM 分析结果
    weekly_report: WeeklyReport # 周报数据
    alerts: List[AnalyzedItem]  # 需即时推送的高优条目
    errors: List[str]           # 错误日志
    run_id: str                 # 本次运行唯一 ID
    started_at: datetime        # 启动时间

两种运行模式

weekly_report 模式
每周一 09:00 自动触发，采集一周内所有竞品动态， 经 LLM 分析后生成 Markdown 周报，推送到工作群。
realtime_alert 模式
每 2 小时轮询一次，检测到新内容后即时分析， 发现高优先级动态立即生成简报推送到群中。
Technology Stack

技术选型与框架

🔗
LangGraph
Agent 工作流引擎

使用 StateGraph 定义采集→去重→分析→报告→推送的有向图工作流。 支持条件分支（高优→即时推送）、并行执行、状态持久化。 相比纯函数式 pipeline，LangGraph 提供更好的可调试性和可扩展性。

🤖
LangChain + OpenAI API
LLM 调用层

通过 langchain-openai 的 ChatOpenAI 类调用任意 OpenAI 兼容 API。 用户只需配置 LLM_BASE_URL 和 LLM_API_KEY 即可接入自选模型。

⏰
APScheduler
定时任务调度

基于 AsyncIOScheduler 实现异步定时任务。 CronTrigger 用于周报（每周一 09:00）， IntervalTrigger 用于实时监控轮询（每 2 小时）。 支持热重载和任务持久化。

TikHub Integration

请求流程

# TikHubClient._request()
GET {base_url}{endpoint}?{params}

Headers:
  Authorization: Bearer {TIKHUB_API_TOKEN}

限流: Semaphore(qps=10)
重试: @retry(stop=3, wait=exp(2,20))

数据流转全景

LLM 分析 Prompt 设计

System Prompt：定义为专业竞品分析师角色，要求以 JSON 格式返回结构化分析结果，包含 content_type、priority、summary、key_signals、potential_impact、recommended_actions 六个字段。

User Prompt：注入竞品名称、来源渠道、标题、内容摘要作为上下文。

降级策略：LLM 调用失败时自动切换为 fallback 分析（仅提取基础信息，标记为待人工审核）。

消息推送架构

企业微信：通过 Webhook 发送 Markdown 消息，支持 4096 字长文本自动截断。

飞书：通过 Interactive Card 格式发送，支持富文本卡片样式。

钉钉：通过加签 Webhook 发送，支持 HMAC-SHA256 安全签名。

CompositeNotifier 自动检测已配置的渠道，同时向所有启用渠道推送。

Quick Start

快速启动指南

1

安装依赖

cd competitor-monitor
pip install -r requirements.txt
2

配置环境变量

cp .env.example .env
# 编辑 .env 填入：
#   LLM_API_KEY    ← 你的大模型 API Key
#   LLM_BASE_URL   ← API 地址
#   TIKHUB_API_TOKEN ← TikHub 平台 Token
#   WECHAT_WORK_WEBHOOK_URL ← 企微机器人地址
3

运行

# 手动生成周报
python main.py weekly 腾讯WorkBuddy

# 手动执行一次监控
python main.py monitor Claude

# 启动定时调度服务
python main.py serve
