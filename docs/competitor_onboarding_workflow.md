# 竞品监测目标接入工作流

## 1. 明确产品身份

1. 确认产品正式名称、所属公司、核心定位和目标用户。
2. 对同名产品至少使用公司官网、官方公告或官方账号中的两项交叉验证。
3. 记录产品别名、中英文名、旧名称和容易产生噪声的歧义词。

## 2. 收集官方信源

按以下优先级记录可持续监测的页面：

1. 产品官网或产品页。
2. 官方发布、博客、新闻中心、更新日志。
3. 使用文档、帮助中心、开发者文档。
4. 官方 GitHub 组织与产品仓库。
5. 官方公司公告中明确关联的其他信源。

只有能够确认官方归属的仓库才使用 `github_repo_source`；第三方项目使用带产品限定词的 `github_search_source`。

## 3. 设计检索词

每个产品维护 3 到 6 个能够独立检索的短语：

- 正式产品名。
- 中文名或英文名。
- 关键子产品、功能名或旧名称。
- 公司名加产品名，用于消除歧义。

避免单独使用 `AI`、`Code`、`Work` 等高噪声通用词。国内外平台共用关键词时，应同时包含中文语境和英文语境下的常见写法。

## 4. 核验社交账号

逐个平台检查 X、YouTube、Reddit、Instagram、Threads、TikTok、微博、小红书、知乎、Bilibili、抖音：

1. 优先使用官网页脚或官方公告链接到的账号。
2. 其次使用已认证账号简介中反向链接到官网的账号。
3. 无法确认时填 `—`，不要用搜索结果中的同名账号替代。
4. 记录账号 handle、频道名、UID 或主页 URL；只有显示名时采集器会退化为关键词搜索。
5. 关键人物只收录创始人、CEO、产品负责人或官方开发者关系负责人，并记录证据与角色。

## 5. 修改配置和账号表

1. 在 `config/competitors.py` 添加 `CompetitorConfig`。
2. 同时配置官网、文档、国内社交平台、海外社交平台和 GitHub 信源。
3. 在 `files/social_media_accounts.xlsx` 的“竞品官方账号总览”中增删产品行。
4. 同步维护“关键人物账号”，并重新生成“平台覆盖统计”。
5. 产品名称必须与 `CompetitorConfig.name` 完全一致，否则账号采集器无法关联。

## 6. 验收

1. 校验产品名无重复，已删除产品不再出现在配置和账号表中。
2. 校验所有网页 URL、GitHub owner/repo 格式和社交平台名称。
3. 验证每个新增产品至少有官网、国内社交搜索、海外社交搜索三个信号层。
4. 运行配置、账号映射、关键词展开和编译测试。
5. 网络不可用时明确标注未在线验证的 URL 或账号，不把推测信息写成已确认事实。
